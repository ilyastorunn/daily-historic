## App Icon

### Technical Requirements

| Platform | Size | Format | Notes |
|----------|------|--------|-------|
| Apple App Store | 1024x1024px | PNG, no alpha channel | Required for App Store Connect upload |
| Google Play Store | 512x512px | PNG, 32-bit with alpha | Required for Play Console upload |
| iOS App Bundle | 60x60px, 120x120px, 180x180px | PNG | Auto-generated by Xcode from 1024px master |
| Android Adaptive | 108x108px foreground layer | PNG | Safe zone: center 72x72px |

### Design Direction

**Concept:** The icon should communicate "history" and "discovery" instantly at 60x60px. Avoid text — it becomes illegible at small sizes.

**Recommended approach:** A stylized hourglass, calendar, or clock face integrated with a subtle antique or parchment aesthetic. The "H" letterform from Historiq could be incorporated as a secondary element but must not be the primary recognition anchor at small sizes.

**Color palette:**
- Primary: Deep navy or charcoal (#1A1F35 range) — conveys depth, intellect, timelessness
- Accent: Warm gold or amber (#C8962E range) — communicates history, discovery, premium feel
- Avoid: Bright red or generic blue (overused in education category)

**Competitor differentiation:**
- History Channel app: Red/white block logo — avoid red
- This Day in History: Calendar icon — differentiate with non-calendar treatment
- Timehop: Illustrated cartoon style — lean more refined/intellectual

**Testing note:** Icon is the highest-leverage A/B test asset. Plan two variants from the start (see ab-test-setup.md).

## Screenshot Content Strategy

### Screenshot 1 — Hero (Most Critical)
**Purpose:** Immediate value proposition. This is what users see in search before tapping.

**Content:** Show the home feed with 2–3 beautifully rendered historical event cards. Include a text overlay that states the primary value prop.

**Recommended overlay text:** "History, personalized for you." or "What happened on this day — in your eras."

**Visual treatment:** Full-bleed screenshot with a subtle gradient overlay at top/bottom for text legibility. Dark mode preferred (looks premium in search results).

### Screenshot 2 — Time Machine Feature
**Purpose:** Demonstrate the unique differentiator that no competitor offers.

**Content:** Show the Time Machine date picker in use, with a dramatic historical event result visible.

**Recommended overlay text:** "Travel to any date in history." or "The Time Machine. Go anywhere."

### Screenshot 3 — Personalization / Onboarding
**Purpose:** Communicate that the app adapts to the user — this reduces friction for first-time visitors who worry the content will be generic.

**Content:** Show the era/category selection screen from onboarding, or the resulting personalized feed.

**Recommended overlay text:** "Your interests. Your history." or "Choose your eras. Build your feed."

### Screenshot 4 — Explore & Search
**Purpose:** Show content depth. Users want to know there's more than a daily feed.

**Content:** Explore screen with a search query in progress and filtered results showing variety.

**Recommended overlay text:** "5,000 years. Searchable." or "Every era. Every category."

### Screenshot 5 — Event Detail
**Purpose:** Show content quality. Users on the fence about installing often scroll to see what an actual event looks like.

**Content:** A rich event detail view — a well-known event with strong visual context. Choose something universally recognizable (Moon Landing, fall of Rome, WWII).

**Recommended overlay text:** Optional — let the content speak for itself here.

## Text Overlay Guidelines

- Font size: 28pt minimum for overlay text (legibility at thumbnail size in search)
- Contrast: All text must pass WCAG AA (4.5:1 ratio minimum)
- Max 8 words per overlay — users spend under 1 second per screenshot in browse mode
- Use the app's existing typography system for brand consistency
- Avoid "Download now" or "Free" overlays — Apple prohibits pricing claims in screenshots; Google discourages generic CTAs

## Video Preview (Strongly Recommended)

### Apple App Store Preview
- Duration: 15–30 seconds (auto-plays muted in search)
- Resolution: 886x1920px (portrait) or 1920x886px (landscape)
- Format: H.264 or HEVC
- Auto-play starts immediately — first 3 seconds must be compelling with no intro/title card
- No audio required; if included, must work without sound

**Recommended flow (30 seconds):**
1. 0–5s: App icon animates in → home feed loads with personalized events
2. 5–12s: User scrolls feed, taps an event → rich event detail view
3. 12–20s: Time Machine feature — date picker spins, dramatic event appears
4. 20–27s: Explore screen with search and filters in use
5. 27–30s: App icon + "Download Historiq" end card

### Google Play Store Feature Graphic
- Size: 1024x500px (landscape banner, shown at top of Play Store listing)
- Format: PNG or JPEG
- This is not a video thumbnail — it appears as a standalone banner when no video is present
- Content: App name + tagline + one key visual (e.g., stylized event card or Time Machine UI)

## Localization Notes

All screenshot text overlays should be in English (U.S.) for the initial launch. If localization is planned, create separate screenshot sets per locale — do not machine-translate overlays, as educational app copy requires native review.
