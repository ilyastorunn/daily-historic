# Ongoing ASO and Optimization Tasks - Historiq

**Last Updated:** March 25, 2026
**Status:** Pre-launch (activate schedule on May 20, 2026 launch day)

---

## Daily Tasks (15-20 minutes/day)

### Morning Check (9:00 AM)
- [ ] Open App Store Connect app -- check overnight reviews
- [ ] Open Google Play Console app -- check overnight reviews
- [ ] Respond to any critical reviews (crashes, data loss, security)
- [ ] Check crash reports: Firebase Crashlytics dashboard
- [ ] Quick glance at download numbers -- any sudden drops or spikes?

### End of Day Check (5:00-6:00 PM)
- [ ] Respond to remaining reviews (goal: 100% within 24 hours)
- [ ] Check if any review is gaining traction (many "helpful" votes)
- [ ] Note today's download count for weekly tracking

**Time commitment:** 15-20 minutes total
**Tools:** App Store Connect app (iOS), Google Play Console app (Android), Firebase Console

---

## Weekly Tasks (1 hour, every Monday morning)

### Keyword Rankings Check
- [ ] Search App Store for top 5 target keywords -- note Historiq's position
- [ ] Search Google Play for top 5 target keywords -- note position
- [ ] Record in tracking spreadsheet
- [ ] Identify any keywords that dropped significantly (> 10 positions)
- [ ] Identify any new keyword opportunities from competitor changes

**Target keywords to track:**
1. "today in history"
2. "historical events"
3. "this day in history"
4. "daily history"
5. "on this day"
6. "history app"
7. "time machine history"
8. "personalized history"

### Conversion Rate Analysis
- [ ] Apple: App Store Connect > Analytics > Conversion Rate
- [ ] Google: Play Console > Statistics > Store listing conversion
- [ ] Compare this week vs. last week
- [ ] Note conversion rate by source (search vs. browse vs. external)
- [ ] Flag if CVR drops > 10% week-over-week

### Competitor Monitoring
- [ ] Check top 3 competitors for store listing changes:
  - This Day in History
  - Timehop
  - On This Day / History Channel app
- [ ] Note any new screenshots, description changes, or feature updates
- [ ] Check if new competitors appeared in "today in history" search results

### Review Sentiment Summary
- [ ] Count this week's reviews by rating (5, 4, 3, 2, 1)
- [ ] List top 3 themes from negative reviews
- [ ] List top 3 themes from positive reviews
- [ ] Identify any new feature requests mentioned 3+ times

## Bi-Weekly Tasks (1.5 hours, every other Monday)

### A/B Testing Review
- [ ] Check active A/B tests on Google Play (if running)
  - Google Play supports native A/B testing for icon, screenshots, description
- [ ] Evaluate statistical significance of running tests
- [ ] Implement winning variant if test is conclusive
- [ ] Plan next A/B test hypothesis

### Screenshot and Visual Asset Review
- [ ] Are screenshots still accurate? (If app UI changed in recent update)
- [ ] Is the first screenshot still the strongest hook?
- [ ] Any seasonal opportunity? (e.g., Presidents Day, World War anniversaries, cultural months)

### Promotional Text Update (Apple)
- [ ] Update promotional text if relevant (no app review needed)
- [ ] Tie to seasonal events, new features, or trending historical topics
- [ ] Examples:
  - "March: Discover Women's History Month events personalized for you"
  - "New: Time Machine now covers 3000+ years of history"
  - "Just added: World War II deep dives with rare photos"

## Metrics Dashboard

### Key Metrics to Track Weekly

**Apple App Store (App Store Connect > Analytics):**
- Impressions (total and by source: search, browse, referrer)
- Product page views
- App units (first-time downloads)
- Conversion rate (impressions to installs)
- Average rating
- Number of ratings and reviews
- Crash rate

**Google Play (Play Console > Statistics):**
- Store listing visitors
- Store listing acquisitions (installs)
- Conversion rate
- Ratings distribution
- Reviews
- Crashes and ANRs
- Uninstall rate

**Third-Party (Optional):**
- Keyword rankings and visibility score (AppTweak, Sensor Tower, or manual)
- Download estimates for competitors
- Category ranking

## Automation Opportunities

**Can be automated:**
- Download and rating tracking via App Store Connect API / Play Console API
- Crash alerting via Firebase Crashlytics (email/Slack alerts)
- Review notification via App Store Connect / Play Console mobile apps
- Keyword ranking tracking via third-party tools (AppTweak, data.ai)

**Should remain manual:**
- Review responses (personal touch makes a difference)
- Competitor analysis (requires strategic interpretation)
- Metadata writing and optimization (creativity + data)
- A/B test hypothesis formation
- Seasonal content planning

## Reminder: ASO is Continuous

ASO is not a launch-day activity. The apps that rank well are the ones that optimize consistently. A 30-minute daily habit and 1-hour weekly review will compound into significant ranking and download improvements over 6-12 months.

**The schedule above represents approximately:**
- 2-3 hours/week ongoing
- 4-6 hours/quarter for deep analysis
- 8-10 hours/year for annual audit

This is manageable for a solo developer and will meaningfully impact discoverability and growth.
